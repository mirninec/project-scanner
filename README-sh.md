# 🧠 Project Scanner — генератор анализа проекта для LLM

Скрипт `project_scanner.sh` предназначен для автоматического анализа исходного кода проекта и формирования единого Markdown-файла, оптимизированного для передачи в LLM (например, ChatGPT) для последующего анализа.

---

## 🚀 Что делает скрипт

Скрипт:

📁 Сканирует структуру проекта  
🌳 Строит дерево файлов  
📄 Извлекает содержимое исходного кода  
🚫 Исключает ненужные директории и файлы  
📊 Считает статистику проекта (опционально)  
🧾 Формирует единый файл `project_analysis.md`  

---

## 🧩 Основная цель

👉 Подготовить проект к анализу LLM:

- код-ревью
- архитектурный анализ
- поиск проблем
- миграции
- документирование
- рефакторинг

---

## ⚙️ Установка

Сделайте скрипт исполняемым:

```bash
chmod +x project_scanner.sh
````

---

## ▶️ Использование

### 📌 Базовый запуск

```bash
./project_scanner.sh
```

👉 Сканирует текущую директорию и создаёт файл `project_analysis.md`

---

### 📂 Указать директорию проекта

```bash
./project_scanner.sh /path/to/project
```

---

### 📄 Указать выходной файл

```bash
./project_scanner.sh -o analysis.md
```

---

### 🌳 Только структура проекта

```bash
./project_scanner.sh --tree-only
```

---

### 🚫 Без содержимого файлов

```bash
./project_scanner.sh --no-content
```

---

### 📊 Показать статистику проекта

```bash
./project_scanner.sh --stats
```

---

### 🚫 Исключение файлов

```bash
./project_scanner.sh -f ".env config.local README.md"
```

---

### 📑 Исключения из файла

```bash
./project_scanner.sh --exclude-file-list exclude.txt
```

Формат файла `exclude.txt`:

```
.env
config.local.js
*.log
temp/*
```

---

## 📦 Что попадает в отчёт

Скрипт формирует Markdown-файл со следующими секциями:

### 🧾 1. Метаданные проекта

* имя проекта
* путь
* дата сканирования

---

### 🌳 2. Структура проекта

Дерево директорий и файлов

---

### 📊 3. Статистика (если включено)

* количество файлов
* количество строк кода
* распределение по языкам

---

### 📄 4. Содержимое файлов

Каждый файл отображается как:

```typescript
// пример
const hello = "world";
```

---

### 🚫 5. Исключённые файлы

Отображаются файлы, которые были пропущены фильтрами

---

## 🚫 Что исключается по умолчанию

### 📁 Директории:

* node_modules
* dist
* build
* .git
* .venv
* target
* .idea

### 📄 Файлы:

* package-lock.json
* .gitignore
* README.md
* .env
* и другие системные/служебные файлы

---

## 🧠 Особенности

✨ Поддержка множества языков:

* Python, JS, TS, PHP, Go, Rust, Java, C++, C#, etc.

⚡ Оптимизация под LLM:

* Markdown формат
* code blocks с языками
